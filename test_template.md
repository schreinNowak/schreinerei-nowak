# Test Template - Bitte ausfüllen

Füllen Sie für jeden durchgeführten Test die folgenden Informationen aus:

| Level | V&V Measure / Scope | Geplanter Testumfang | Durchführung | Status | Nachweis (Link) | Anmerkung |
|-------|---------------------|---------------------|--------------|--------|-----------------|-----------|
| I. Requirement Elicitation / System Validation | Acceptance Test | Kunden-/Abnahmetest der Applikation | EPV1 | _ROT_ | _LINK_ | _BEMERKUNG_ |
| II. System Architecture / System Integration & Verification | Tests im Systemversuch | Tests mit realer Hardware im Systemversuch | EDT2 | _GRÜN_ | _LINK_ | _BEMERKUNG_ |
| III. Subsystem Architecture / Subsystem Integration & Verification | Software Function Test | Manuelle Tests nach Test-Spezifikationen | ESW4 | _GRÜN/_ | _LINK_ | _BEMERKUNG_ |
| IV. Component Design and Implementation | Unit Test | Automatische Unit-Tests | ESW4 | _GRÜN/GELB/ROT_ | _LINK_ | _BEMERKUNG_ |
| IV. Component Design and Implementation | Static Code Analysis | Automatische Code-Analyse (z.B. SonarQube) | ESW4 | _GRÜN_ | _LINK_ | _BEMERKUNG_ |
| IV. Component Design and Implementation | Integration Test | Komponenten-Integrationstests | ESW4 | _GRÜN_ | _LINK_ | _BEMERKUNG_ |
| IV. Component Design and Implementation | E2E Test | End-to-End Testszenarien | ESW4 | _GRÜN_ | _LINK_ | _BEMERKUNG_ |

## Legende
- **Durchführung**: Name des Teams/Abteilung (z.B. ESW4, BGSW)
- **Status**: GRÜN (bestanden), GELB (Befunde), ROT (fehlgeschlagen), oder leer lassen wenn nicht durchgeführt
- **Nachweis**: UNC-Pfad oder Link zum Testprotokoll
- **Anmerkung**: Zusätzliche Hinweise, Kommentare
